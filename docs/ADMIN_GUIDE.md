## 1. 登录管理后台

### 访问地址
- 管理后台地址：`http://your-server:8000/admin`
- API文档地址：`http://your-server:8000/docs`

### 登录账号
首次登录使用默认管理员账号：
- **用户名(手机号)**：`admin`
- **密码**：`admin123`（首次登录后请立即修改）

### 登录流程
1. 打开管理后台登录页面
2. 输入手机号和密码
3. 点击"登录"按钮
4. 首次登录后建议修改密码

## 2. 学校管理

### 功能说明
学校管理用于创建和管理不同的项目/部门，每个学校可以有独立的数据库配置和查询模板。

### 创建学校
1. 登录管理后台
2. 进入"学校管理"菜单
3. 点击"新增学校"按钮
4. 填写学校信息：
   - **学校名称**：如"XX中学"、"XX公司"
   - **学校编码**：唯一标识，如`school_001`
   - **描述**：学校简要说明
   - **状态**：启用/停用
5. 点击"保存"按钮

### 编辑学校
1. 在学校列表中找到目标学校
2. 点击"编辑"按钮
3. 修改相关信息
4. 点击"保存"按钮

### 删除学校
1. 在学校列表中找到目标学校
2. 点击"删除"按钮
3. 确认删除操作（删除学校将同时删除相关的数据库配置和查询模板）

## 3. 数据库配置

### 功能说明
配置需要连接的外部数据库，包括MySQL、Oracle、SQL Server等。

### 添加数据库配置
1. 进入"数据库配置"菜单
2. 选择目标学校
3. 点击"新增配置"按钮
4. 填写数据库连接信息：

#### MySQL配置示例
| 字段 | 说明 | 示例值 |
|------|------|--------|
| 配置名称 | 显示名称 | 生产数据库 |
| 数据库类型 | 选择MySQL | MySQL |
| 主机地址 | 数据库服务器IP | 192.168.1.100 |
| 端口 | 数据库端口 | 3306 |
| 用户名 | 数据库用户名 | root |
| 密码 | 数据库密码 | ***** |
| 数据库名 | 要连接的数据库 | school_db |

#### Oracle配置示例
| 字段 | 说明 | 示例值 |
|------|------|--------|
| 配置名称 | 显示名称 | Oracle生产库 |
| 数据库类型 | 选择Oracle | Oracle |
| 主机地址 | 数据库服务器IP | 192.168.1.101 |
| 端口 | 数据库端口 | 1521 |
| 服务名 | Oracle服务名 | ORCL |
| 用户名 | 数据库用户名 | system |
| 密码 | 数据库密码 | ***** |

#### SQL Server配置示例
| 字段 | 说明 | 示例值 |
|------|------|--------|
| 配置名称 | 显示名称 | SQLServer库 |
| 数据库类型 | 选择SQLServer | SQLServer |
| 主机地址 | 数据库服务器IP | 192.168.1.102 |
| 端口 | 数据库端口 | 1433 |
| 用户名 | 数据库用户名 | sa |
| 密码 | 数据库密码 | ***** |
| 数据库名 | 要连接的数据库 | master |
| ODBC驱动 | SQL Server驱动 | ODBC Driver 17 for SQL Server |

### 测试连接
填写完配置后，点击"测试连接"按钮验证配置是否正确。

### 管理配置
- **启用/停用**：控制该配置是否可用
- **编辑**：修改配置信息
- **删除**：删除配置（注意：删除后无法恢复）

## 4. 查询模板管理

### 功能说明
创建和维护查询模板，定义可执行的SQL查询和查询参数。

### 创建查询模板
1. 进入"查询模板"菜单
2. 选择目标学校
3. 点击"新增模板"按钮
4. 填写模板信息：

#### 基本信息
| 字段 | 说明 | 示例 |
|------|------|------|
| 业务大类 | 查询分类编码 | student |
| 大类名称 | 分类显示名称 | 学生信息 |
| 大类图标 | 分类图标(可选) | 👨‍🎓 |
| 查询名称 | 模板显示名称 | 学生列表查询 |
| 描述 | 查询说明 | 查询所有在校学生信息 |

#### SQL模板配置
| 字段 | 说明 | 示例 |
|------|------|------|
| SQL模板 | 查询SQL语句 | SELECT * FROM students WHERE status = :status |
| 时间字段 | 用于时间筛选的字段 | create_time |
| 默认条数限制 | 返回结果上限 | 500 |

#### 参数配置
支持配置查询参数，示例：
```sql
SELECT * FROM students 
WHERE 1=1
{{ AND status = :status }}
{{ AND grade = :grade }}
{{ AND class_id = :class_id }}
{{ AND create_time >= :start_date }}
{{ AND create_time <= :end_date }}
```

参数类型：
- **string**：字符串
- **number**：数字
- **date**：日期
- **select**：下拉选择

### 测试模板
创建模板后可点击"测试"按钮验证SQL是否正确执行。

## 5. 用户管理

### 功能说明
管理系统用户，包括创建用户、分配权限等。

### 创建用户
1. 进入"用户管理"菜单
2. 点击"新增用户"按钮
3. 填写用户信息：
   - **手机号**：登录账号（必须唯一）
   - **姓名**：用户真实姓名
   - **密码**：登录密码
   - **角色**：普通用户/管理员
   - **状态**：启用/停用

### 角色说明
| 角色 | 权限 |
|------|------|
| 管理员(admin) | 完全访问，可管理所有学校和配置 |
| 普通用户(user) | 只能访问被授权的学校 |

### 分配学校权限
1. 在用户列表中找到目标用户
2. 点击"授权"按钮
3. 选择用户可访问的学校
4. 设置具体权限（查询、导出等）
5. 保存授权信息

### 重置密码
管理员可以重置用户密码：
1. 找到目标用户
2. 点击"重置密码"
3. 输入新密码
4. 确认重置

## 6. 日志审计

### 功能说明
记录所有用户的查询操作，便于审计和问题排查。

### 查看日志
1. 进入"日志审计"菜单
2. 可以按以下条件筛选：
   - 时间范围
   - 用户
   - 学校
   - 查询模板
   - 执行状态（成功/失败）

### 日志内容
每条日志记录包含：
- 执行时间
- 执行用户
- 所属学校
- 使用的查询模板
- 查询参数
- 执行的SQL语句
- 返回结果数
- 执行耗时
- 执行状态

### 常见问题排查
1. **查询失败**：查看错误信息列
2. **查询慢**：查看执行耗时
3. **数据异常**：查看执行的SQL和参数

## 常见问题

### Q1: 如何修改管理员密码？
A: 登录后进入"个人中心" -> "修改密码"

### Q2: 数据库连接测试失败怎么办？
A: 1. 检查网络连通性
   2. 检查数据库服务器是否允许远程连接
   3. 检查用户名密码是否正确
   4. 检查防火墙端口是否开放

### Q3: 查询模板测试失败怎么办？
A: 1. 检查SQL语法是否正确
   2. 检查表名和字段名是否存在
   3. 检查用户是否有该表的查询权限

### Q4: 如何导出查询日志？
A: 在日志审计页面，可以选择时间范围和筛选条件，点击"导出"按钮导出Excel。
