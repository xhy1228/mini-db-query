## 🔒 严格打包流程

### 打包前检查清单

```
✅ 1. 确认版本号已更新
   - backend/version.py

✅ 2. 确认文件完整性
   - backend/admin/index.html
   - backend/admin/setup.html
   - backend/requirements.txt
   - start.bat
   - install.bat

✅ 3. 确认编码正确
   - 所有 .bat 文件纯英文
   - requirements.txt 英文注释

✅ 4. 确认功能正常
   - 本地启动测试通过
   - 页面可访问
```

### 打包后验证

```
✅ 1. 解压后目录结构检查
✅ 2. 关键文件存在性检查
✅ 3. 版本号核对
```

## 📝 升级说明规范

每个版本的 Release 必须包含：

### 1. 版本信息
```
## vX.X.X.X - 发布日期

### 新增功能
- [ ] 功能1描述

### 问题修复
- [ ] 问题1描述

### 优化改进
- [ ] 优化1描述
```

### 2. 升级操作说明
```
## 🚀 升级指南

### 从 vX.X.X.X 升级到 vX.X.X.X

#### 需要替换的文件
| 文件路径 | 操作 |
|---------|------|
| backend/version.py | 替换 |
| backend/main.py | 替换 |
| ... | ... |

#### 配置文件变更
| 配置项 | 原值 | 新值 |
|--------|------|------|
| xxx | xxx | xxx |

#### 数据库变更
如有数据库变更，必须说明：
- SQL 脚本位置
- 执行顺序
- 注意事项
```

## 📄 README.md 维护规范

### 历史记录保留原则

**不删除，只追加！**

```markdown
# 项目名称

## 📋 版本历史

### v1.0.0.58 (2026-03-18)
- ✅ 数据库配置页面优化
- ✅ 管理后台版本号显示
- 修复 setup.html 打包遗漏问题

### v1.0.0.57 (2026-03-17)
- 修复根路径返回类型错误

### v1.0.0.56 (2026-03-16)
- 初始版本发布
```

### 增量更新规则

1. **新版本信息追加到顶部**
2. **历史版本记录保留**
3. **不删除任何历史操作说明**
